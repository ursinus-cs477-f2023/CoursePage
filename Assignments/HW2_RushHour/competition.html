Below are the results on the 40 puzzles, sorted in descending order of number of average nodes expanded over all the puzzles. People who broke consistency and hence ended up with a non-optimal length of steps from start to finish are disqualified, so Kevin is the winner!

I bolded the number of expansions if that particular entry got the best score on a particular puzzle. I crossed out the number of steps if that particular entry did not compute an optimal sequence from the start to the goal. A few observations:

  • Greedy best-first blows other techniques out of the water in terms of number of expansions needed. But it's not guaranteed to return an optimum
  • The best performing techniques all did some combination of distance of goal car to goal + number of cars blocking. Note that the number of cars blocking is stronger than the blocking heuristic.
  • No two people had exactly the same scores across all puzzles. Even techniques that did very similar things ended with different scores. This shows how very subtle changes (e.g. the order in which neighbors were expanded) can affect the results. This is why we have to be very careful when drawing conclusions about what changes in AI/ML pipelines have led to improvements, and why we need good controls and many experiments!
